# Particle-Trajectory-Simulator
A high-performance C++ simulation designed to model the movement of charged particles within non-uniform magnetic fields. This project demonstrates the application of numerical methods to solve fundamental equations of motion in classical electromagnetism.
Technical Analysis: 4th Order Runge-Kutta Particle Trajectory Simulator🔬 Scientific AbstractThis repository houses a high-fidelity computational engine designed to simulate the dynamics of charged particles within electromagnetic fields. By solving the Lorentz Force differential equations, the simulator provides a precise mapping of particle drift, helical motion, and cyclotron resonance.The primary objective of this work is to demonstrate the application of Object-Oriented C++ in solving complex ordinary differential equations (ODEs) that are fundamental to High Energy Physics (HEP) research and accelerator design.📐 Numerical Methodology: The RK4 AdvantageStandard integration techniques, such as the Euler or Improved Euler (Heun's) method, often suffer from cumulative energy drift in periodic systems. This simulator implements the 4th Order Runge-Kutta (RK4) Algorithm, which offers a superior balance between computational cost and a local truncation error of $O(h^5)$.The Integration LogicFor a state vector $y = [r, v]$, the algorithm computes four distinct increments:$k_1$: The slope at the beginning of the interval.$k_2$: The slope at the midpoint, using $k_1$ to sample the field.$k_3$: The slope at the midpoint, using $k_2$ for refined sampling.$k_4$: The slope at the end of the interval.The final state is a weighted average of these increments, ensuring the simulation maintains a stable orbit even over extended time-steps ($dt$), where simpler methods would spiral outward due to numerical instability.💻 System ArchitectureThe project is structured as a modular data pipeline to mimic professional research environments:I. The Computational Engine (C++)Encapsulation: Physics constants (charge-to-mass ratio) and solver parameters are encapsulated within a modular ParticleTracker class.Vector Algebra: Custom operator overloading for 3D vectors ensures code readability and mathematical transparency.Data Integrity: High-precision floating-point arithmetic is utilized for coordinate logging to trajectory_data.csv.II. The Analytical Layer (Python)Decoupled Visualization: By separating the math engine from the UI, the system remains lightweight and portable across Linux and Windows environments.Robust Data Parsing: The script handles CSV parsing with automated header cleaning and relative path resolution, eliminating environment-specific configuration errors.📊 Simulation Results & ObservationThe simulator successfully reproduces the Helical Trajectory expected in a uniform magnetic field. Key observations include:Cyclotron Frequency: The simulated period accurately matches the theoretical $T = 2\pi m / qB$.Energy Conservation: The magnitude of the velocity vector remains constant (within $10^{-6}$ precision), confirming the RK4 solver's stability under Lorentz forces where $v \cdot F = 0$.🎓 About the ProjectDeveloped by Hiten Kaistha as an undergraduate research project at NIT Hamirpur, exploring the intersection of Photonic Sciences and Computational Physics.
